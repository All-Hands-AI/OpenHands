## 🛠️ للبدء

أفضل طريقة لتشغيل OpenHands هي داخل حاوية Docker. يعمل البرنامج بشكل أفضل مع أحدث إصدار من Docker، `26.0.0`.  
يجب عليك استخدام Linux أو Mac OS أو WSL على Windows.

لتشغيل OpenHands داخل حاوية Docker، نفذ الأوامر التالية في الطرفية:

:::warning
عند تنفيذ الأمر التالي، قد يتم تعديل الملفات داخل `./workspace` أو حذفها.
:::

```bash
WORKSPACE_BASE=$(pwd)/workspace
docker run -it \
    --pull=always \
    -e SANDBOX_USER_ID=$(id -u) \
    -e WORKSPACE_MOUNT_PATH=$WORKSPACE_BASE \
    -v $WORKSPACE_BASE:/opt/workspace_base \
    -v /var/run/docker.sock:/var/run/docker.sock \
    -p 3000:3000 \
    --add-host host.docker.internal:host-gateway \
    --name openhands-app-$(date +%Y%m%d%H%M%S) \
    ghcr.io/all-hands-ai/openhands:main
```

ستجد OpenHands قيد التشغيل على الرابط التالي: [http://localhost:3000](http://localhost:3000)، مع إمكانية الوصول إلى `./workspace`.  
لتعمل OpenHands على مشروعك الخاص، ضع الكود داخل مجلد `./workspace`.

سيقتصر وصول OpenHands على هذا المجلد فقط، حيث يتم تشغيله داخل بيئة معزولة وآمنة باستخدام Docker.

:::tip
إذا كنت ترغب في استخدام الإصدار الأحدث **(غير مستقر!)**، يمكنك استخدام `ghcr.io/all-hands-ai/openhands:main` كصورة (آخر سطر).
:::

للتعرف على سير العمل الخاص بالتطوير، راجع [Development.md](https://github.com/All-Hands-AI/OpenHands/blob/main/Development.md).

**هل تواجه مشاكل؟**  
اطلع على [دليل استكشاف الأخطاء وإصلاحها](https://docs.all-hands.dev/modules/usage/troubleshooting).

:::warning
OpenHands قيد التطوير حاليًا، لكن يمكنك تشغيل النسخة الأولية (Alpha) لمشاهدة النظام يعمل بشكل كامل.
:::
