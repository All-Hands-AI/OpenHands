## üöÄ Getting Started

Before you begin, ensure you have the following:

- Docker (version 26.0.0 or later recommended)
- Operating System: Linux, macOS, or Windows with WSL
- Minimum 8GB RAM and 4 CPU cores (16GB RAM and 8 cores recommended)

## üõ†Ô∏è Installation

<Tabs>
  <TabItem value="step1" label="Step 1" default>

  Create a workspace directory:

  <CodeBlock language="bash">
  mkdir workspace
  </CodeBlock>

  </TabItem>
  <TabItem value="step2" label="Step 2">

  Run the Docker container:

  <CodeBlock language="bash">
  WORKSPACE_BASE=$(pwd)/workspace
  docker run -it \
      --pull=always \
      -e SANDBOX_USER_ID=$(id -u) \
      -e WORKSPACE_MOUNT_PATH=$WORKSPACE_BASE \
      -v $WORKSPACE_BASE:/opt/workspace_base \
      -v /var/run/docker.sock:/var/run/docker.sock \
      -p 3000:3000 \
      --add-host host.docker.internal:host-gateway \
      --name opendevin-app-$(date +%Y%m%d%H%M%S) \
      ghcr.io/opendevin/opendevin:0.6.2
  </CodeBlock>

  </TabItem>
</Tabs>

:::caution
When you run the Docker command, files in `./workspace` may be modified or deleted.
:::

## üìò Usage

1. Access OpenDevin at [http://localhost:3000](http://localhost:3000)
2. Place your code in the `./workspace` directory
3. Use the web interface to interact with OpenDevin

:::tip
For advanced usage, check our [Development Guide](https://github.com/OpenDevin/OpenDevin/blob/main/Development.md).
:::

## üîß Troubleshooting

<details>
  <summary>Docker container won't start</summary>
  Ensure Docker is running and you have sufficient system resources.
</details>

<details>
  <summary>Can't access localhost:3000</summary>
  Check if port 3000 is not in use by another application.
</details>

<details>
  <summary>Workspace not updating</summary>
  Verify that you've mounted the correct local directory.
</details>

For more help, visit our [Troubleshooting Guide](https://opendevin.github.io/OpenDevin/modules/usage/troubleshooting).
